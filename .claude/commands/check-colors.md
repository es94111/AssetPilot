# 配色審查與一致性檢查

審查網頁配色是否一致且具美感，並提出具體改善建議。

## 步驟 1：讀取現有配色

讀取 `style.css`，找出所有顏色定義：
- CSS 變數（`:root { --color-xxx }` 區塊）
- 硬編碼顏色值（`#xxx`、`rgb()`、`hsl()`）
- Chart.js 圖表配色（在 `app.js` 中搜尋）

## 步驟 2：配色一致性檢查

檢查以下項目：

### 色彩系統
- [ ] 是否使用 CSS 變數統一管理主色、輔色、背景色、文字色？
- [ ] 是否有未使用 CSS 變數的硬編碼顏色？

### 語義一致性
- [ ] **收入**：是否統一使用綠色系（`--color-income` 或類似命名）？
- [ ] **支出**：是否統一使用紅色系（`--color-expense` 或類似命名）？
- [ ] **成功/正向**：綠色
- [ ] **警告**：黃/橙色
- [ ] **危險/錯誤**：紅色
- [ ] **主色調（品牌色）**：藍色或自訂主色

### 視覺層次
- [ ] 背景色（最淡）→ 卡片色（次淡）→ 邊框色 → 文字色（最深）是否有明確層次？
- [ ] 深色 sidebar 與淺色主內容區是否有足夠對比？

### 圖表配色
- [ ] 圓餅圖各分類顏色是否與主題色系一致？
- [ ] 折線圖、長條圖的顏色是否與收入/支出語義一致？

## 步驟 3：提出改善建議

針對發現的問題，提供：
1. **問題描述**：哪裡不一致或不美觀
2. **建議修改**：具體的 CSS 變數或色值建議
3. **優先順序**：高（影響整體視覺）/ 中（局部不一致）/ 低（微調）

## 步驟 4：執行修改（需使用者確認）

列出所有建議後，**詢問使用者**是否要執行修改，以及哪些項目要優先處理。

---

**審查標準參考：**
- 主色與輔色建議使用色相差 30-180 度的組合
- 背景與前景文字對比度應符合 WCAG AA 標準（4.5:1）
- 同一功能的顏色在所有頁面保持一致
- 圖表顏色建議使用固定色票，避免隨機產生
