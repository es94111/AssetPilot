## 1. 簡介

### 1.1 目的

本文件定義「資產管理」網頁應用程式的軟體需求規格，作為系統設計、開發與測試的依據。

### 1.2 範圍

本系統為一個網頁應用程式，提供使用者記錄日常收入與支出、管理帳戶轉帳、設定預算、檢視統計報表、管理分類（含子分類）、匯出匯入資料、股票投資組合管理等功能，協助使用者掌握個人財務狀況與投資績效。

### 1.3 名詞定義

| 名詞       | 定義                                         |
| ---------- | -------------------------------------------- |
| 交易記錄   | 一筆收入、支出或轉帳的資料                   |
| 分類       | 交易記錄的類別（如餐飲、交通），支援子分類   |
| 子分類     | 隸屬於父分類的細項分類（如餐飲 > 早餐）      |
| 帳戶       | 資金來源（如現金、銀行帳戶、信用卡）         |
| 預算       | 使用者設定的月度支出上限                     |
| 固定收支   | 週期性自動產生的交易記錄                     |
| 轉帳       | 帳戶間的資金移轉（產生轉出+轉入配對記錄）   |
| 批次操作   | 一次對多筆交易進行刪除或變更                 |
| 股票持倉   | 使用者持有的股票資產記錄                     |
| 股票交易   | 股票買入或賣出的交易記錄                     |
| 股票股利   | 股票現金股利或股票股利的發放記錄             |
| FIFO       | 先進先出法，用於計算股票成本基礎與損益       |
| TWSE       | 臺灣證券交易所（Taiwan Stock Exchange）      |

### 1.4 參考文件

- IEEE 830-1998 軟體需求規格標準

## 2. 整體描述

### 2.1 產品概觀

本系統為獨立的網頁應用程式，使用者透過瀏覽器即可使用。系統採用前後端一體化架構（Node.js + Express + SQLite），支援桌面與行動裝置瀏覽。

### 2.2 技術架構

| 層級   | 技術                                             |
| ------ | ------------------------------------------------ |
| 前端   | 原生 HTML / CSS / JavaScript（單頁應用程式 SPA） |
| 後端   | Node.js + Express                                |
| 資料庫 | SQLite（透過 sql.js）                            |
| 驗證   | JWT（JSON Web Token），有效期 7 天               |
| 密碼   | bcryptjs 加密儲存                                |
| SSO    | Google Identity Services（選配）                 |
| 圖表   | Chart.js                                         |
| 圖示   | Font Awesome 6                                   |
| 外部 API | TWSE OpenAPI（臺灣證券交易所即時股價）          |

### 2.3 使用者角色

| 角色       | 說明                               |
| ---------- | ---------------------------------- |
| 一般使用者 | 註冊帳號後使用所有功能             |
| 訪客       | 未登入狀態，僅可瀏覽登入/註冊頁面 |

### 2.4 操作環境

- **前端：** 現代瀏覽器（Chrome、Firefox、Safari、Edge 最新版本）
- **後端：** Node.js 18+
- **部署：** 本地伺服器或雲端，預設 port 3000

### 2.5 限制條件

- 系統僅支援新台幣（NT$）為預設貨幣
- 不提供銀行帳戶自動連結功能
- 單一使用者帳號不支援多人共用

## 3. 功能需求

### 3.1 使用者管理

#### FR-001 使用者註冊

- **描述：** 使用者可透過電子郵件與密碼建立帳號
- **輸入：** 電子郵件、密碼、顯示名稱
- **處理規則：**
  - 電子郵件格式驗證，不可重複註冊
  - 密碼長度至少 6 字元
- **輸出：** 註冊成功後自動登入，建立預設分類（含子分類）與預設帳戶

#### FR-002 使用者登入 / 登出

- **描述：** 使用者輸入帳號密碼登入系統
- **輸入：** 電子郵件、密碼
- **處理規則：**
  - 驗證帳號密碼是否正確
  - 登入成功後發放 JWT Token，有效期 7 天
  - Token 儲存於 localStorage，重新開啟瀏覽器自動恢復登入
- **輸出：** 登入成功導向主頁面（依 URL 路由）；登出後導向登入頁

#### FR-003 Google SSO 登入（選配）

- **描述：** 使用者可使用 Google 帳號快速登入或註冊
- **前置條件：** 伺服器已設定 `GOOGLE_CLIENT_ID` 環境變數
- **處理規則：**
  - 前端透過 `/api/config` 取得 Google Client ID，動態顯示 Google 登入按鈕
  - 使用者點擊 Google 按鈕，透過 Google Identity Services 取得 ID Token
  - 後端透過 Google `oauth2.googleapis.com/tokeninfo` 驗證 Token 並檢查 audience
  - 若該 Email 已存在帳號，直接登入；若不存在，自動建立帳號（含預設分類與帳戶）
  - Google SSO 建立的帳號密碼欄位設為隨機 hash（不可用密碼登入，僅能用 Google）
- **輸出：** 登入成功導向主頁面，與一般登入相同
- **備註：** 未設定 Client ID 時，Google 按鈕自動隱藏，不影響帳號密碼登入功能

#### FR-004 股票交易紀錄批次刪除與分頁

- **描述：** 股票交易紀錄支援多選批次刪除與分頁瀏覽
- **處理規則：**
  - 每列前方有 checkbox，表頭有全選 checkbox
  - 勾選後顯示批次操作列（已選 N 筆 + 批次刪除按鈕 + 取消按鈕）
  - 每頁筆數可選 10 / 20 / 50 / 100 / 自訂
  - 伺服器端分頁（`page` + `pageSize` 查詢參數）
- **API：**
  - `GET /api/stock-transactions?page=1&pageSize=20` — 回傳 `{ data, total, page, totalPages }`
  - `POST /api/stock-transactions/batch-delete` — 批次刪除（`{ ids: [...] }`）

#### FR-005 股利紀錄批次刪除與分頁

- **描述：** 股利紀錄支援多選批次刪除與分頁瀏覽
- **處理規則：** 同 FR-004
- **API：**
  - `GET /api/stock-dividends?page=1&pageSize=20` — 回傳 `{ data, total, page, totalPages }`
  - `POST /api/stock-dividends/batch-delete` — 批次刪除（`{ ids: [...] }`）

### 3.2 交易記錄管理

#### FR-010 新增交易記錄

- **描述：** 使用者可新增一筆收入或支出記錄
- **輸入：**
  - 類型（收入 / 支出）— 必填
  - 金額 — 必填，正數
  - 日期 — 必填，預設為今天
  - 分類 — 必填（支援子分類，以 optgroup 群組顯示）
  - 帳戶 — 必填
  - 備註 — 選填，最多 200 字
- **處理規則：**
  - 金額不可為 0 或負數
  - 日期不可為未來日期
- **輸出：** 新增成功 Toast 提示，更新交易列表

#### FR-011 編輯交易記錄

- **描述：** 使用者可修改已存在的交易記錄
- **輸入：** 同 FR-010
- **處理規則：** 同 FR-010，記錄最後修改時間
- **輸出：** 修改成功 Toast 提示

#### FR-012 刪除交易記錄

- **描述：** 使用者可刪除交易記錄
- **處理規則：**
  - 刪除前需二次確認（確認 Modal）
  - 若為轉帳交易，同時刪除關聯的 linked 交易
- **輸出：** 刪除成功 Toast 提示，更新交易列表

#### FR-013 檢視交易記錄列表

- **描述：** 使用者可瀏覽交易記錄列表
- **顯示欄位：** 勾選框、日期、類型、分類（含父分類 > 子分類顯示）、金額、帳戶、備註、操作
- **處理規則：**
  - 依日期由新到舊排序
  - 支援分頁，每頁筆數可選：10 / 20 / 50 / 100 / 自訂
  - 類型以色彩標籤顯示（支出紅、收入綠、轉帳藍）

#### FR-014 搜尋與篩選交易記錄

- **描述：** 使用者可依條件篩選交易記錄
- **篩選條件：**
  - 日期範圍（起始日 ~ 結束日）
  - 類型（收入 / 支出 / 轉帳 / 全部）
  - 分類（以 optgroup 群組顯示子分類）
  - 帳戶
  - 關鍵字搜尋（備註欄位）
  - 每頁筆數（10 / 20 / 50 / 100 / 自訂輸入）
- **操作：** 篩選按鈕套用、重置按鈕恢復預設

#### FR-015 批次操作

- **描述：** 使用者可多選交易記錄進行批次操作
- **功能：**
  - 勾選框：每列左側 checkbox，表頭有全選 checkbox（支援全選/半選/取消）
  - 批次操作列：勾選後顯示紫色操作列，顯示已選筆數
  - **批次刪除：** 一次刪除多筆交易（含轉帳關聯刪除），需二次確認
  - **批次變更分類：** 一次將多筆交易變更為指定分類
  - **批次變更帳戶：** 一次將多筆交易變更為指定帳戶
  - **批次變更日期：** 一次將多筆交易變更為指定日期
  - **取消選取：** 清除所有勾選，隱藏操作列

### 3.3 帳戶間轉帳

#### FR-016 新增轉帳記錄

- **描述：** 使用者可記錄帳戶間的資金移轉
- **輸入：** 轉出帳戶、轉入帳戶、金額、日期、備註
- **處理規則：**
  - 自動產生一對 transfer_out 和 transfer_in 交易記錄
  - 兩筆記錄透過 linked_id 互相關聯
  - 刪除其中一筆時同步刪除另一筆

### 3.4 分類管理

#### FR-020 檢視分類列表

- **描述：** 使用者可檢視所有收支分類，以階層式結構顯示
- **預設支出分類及子分類：**
  - 餐飲：早餐、午餐、晚餐、飲料、點心
  - 交通：公車/捷運、計程車/Uber、加油、停車費、高鐵/火車
  - 購物：日用品、服飾、3C產品、家電
  - 娛樂：電影/影音、遊戲、旅遊、運動健身
  - 居住：房租/房貸、水電費、網路費、管理費
  - 醫療：掛號費、藥品、保健食品
  - 教育：學費、書籍、線上課程
  - 其他
- **預設收入分類：** 薪資、獎金、投資、兼職、其他
- **顯示方式：**
  - 父分類顯示完整寬度，帶 `+`（新增子分類）和編輯按鈕
  - 子分類以網格佈局縮排顯示，帶左側藍色邊框和箭頭圖示
  - 非預設分類可刪除

#### FR-021 新增分類

- **描述：** 使用者可新增頂層分類或子分類
- **輸入：** 分類名稱、類型（收入/支出）、顏色、上層分類（可選）
- **處理規則：**
  - 同父分類下名稱不可重複
  - 子分類的類型必須與父分類相同
  - 子分類僅支援一層（不可建立孫分類）

#### FR-022 編輯分類

- **描述：** 使用者可修改分類名稱與顏色
- **處理規則：**
  - 若為父分類且有子分類，不可變更為子分類
  - 預設分類不可刪除

#### FR-023 刪除分類

- **描述：** 使用者可刪除非預設分類
- **處理規則：**
  - 該分類下若有交易記錄，不可刪除
  - 刪除父分類時，若子分類下有交易記錄，不可刪除
  - 刪除父分類時連帶刪除所有子分類

#### FR-024 預設子分類自動建立

- **描述：** 新使用者註冊時自動建立完整的預設分類與子分類
- **處理規則：** 既有使用者在系統升級後，自動補建缺少的預設子分類

### 3.5 帳戶管理

#### FR-030 新增帳戶

- **描述：** 使用者可新增資金帳戶
- **輸入：** 帳戶名稱、初始餘額、圖示
- **預設帳戶：** 現金

#### FR-031 編輯帳戶

- **描述：** 使用者可修改帳戶名稱、圖示

#### FR-032 刪除帳戶

- **描述：** 使用者可刪除帳戶
- **處理規則：** 帳戶下若有交易記錄，不可刪除

#### FR-033 檢視帳戶餘額

- **描述：** 顯示各帳戶目前餘額與總資產
- **計算方式：** 初始餘額 + 收入總額 - 支出總額 + 轉入金額 - 轉出金額

### 3.6 預算管理

#### FR-040 設定月度預算

- **描述：** 使用者可設定每月總支出預算或分類預算
- **輸入：** 預算金額、適用月份、分類（可選，不選為總預算）
- **分類選擇：** 支援子分類群組顯示

#### FR-041 預算進度顯示

- **描述：** 儀表板顯示當月預算使用進度
- **顯示內容：**
  - 已使用金額 / 預算金額
  - 使用百分比進度條
  - 剩餘可用金額
  - 超支時進度條變紅色

### 3.7 統計報表

#### FR-050 儀表板收支總覽

- **描述：** 以儀表板形式顯示財務摘要
- **顯示內容：**
  - 本月收入總額、本月支出總額、淨收支
  - 支出分類圓餅圖（Doughnut Chart）
  - 預算進度
  - 近期交易記錄（最近 5 筆）

#### FR-051 分類統計圖表

- **描述：** 以圓餅圖（Doughnut Chart）顯示各分類支出/收入佔比
- **顯示：** 各分類金額、百分比
- **操作：** 可切換期間與類型

#### FR-052 趨勢分析圖表

- **描述：** 以折線圖顯示每日收支趨勢
- **操作：** 可切換期間與類型

#### FR-053 每日消費圖表

- **描述：** 以長條圖顯示每日支出金額
- **操作：** 可切換期間與類型

#### FR-054 報表期間選擇

- **描述：** 統計報表支援多種期間選擇
- **預設選項：** 本月、上月、近 3 個月、近 6 個月、今年
- **自訂時間：** 選擇「自訂時間」後顯示起始日期與結束日期輸入框，使用者可自由指定任意日期範圍
- **處理規則：**
  - 切換為自訂時間時顯示日期輸入框
  - 切換回預設選項時自動隱藏日期輸入框
  - 日期變更時即時重新繪製報表
  - 僅填起始日時，結束日預設為今天；僅填結束日時，起始日預設為當月1日

#### FR-055 報表類型切換

- **描述：** 可切換顯示支出或收入的統計資料
- **選項：** 支出、收入

### 3.8 固定收支

#### FR-060 新增固定收支

- **描述：** 使用者可設定週期性自動產生的交易記錄
- **輸入：** 類型（收入/支出）、金額、分類（支援子分類）、帳戶、備註、起始日期、週期（每日/每週/每月/每年）
- **處理規則：** 系統登入時自動檢查並產生到期的固定收支交易

#### FR-061 管理固定收支

- **描述：** 使用者可檢視、編輯、暫停（啟用/停用切換）、刪除固定收支

### 3.9 資料匯出匯入

#### FR-070 匯出交易記錄

- **描述：** 使用者可將交易記錄匯出為 CSV 檔案
- **格式：** UTF-8 BOM CSV
- **欄位：** 日期、類型、分類（父分類 > 子分類格式）、金額、帳戶、備註
- **篩選：** 可選擇匯出的日期範圍

#### FR-071 匯入交易記錄

- **描述：** 使用者可匯入 CSV 格式的交易記錄
- **格式：** CSV，欄位順序：日期、類型、分類、金額、帳戶、備註
- **支援類型：** 支出、收入、轉出、轉入
- **處理規則：**
  - 第一行為標題列，自動跳過
  - 支援拖曳或點擊上傳
  - 匯入前顯示預覽表格（前 10 筆）
  - 自動偵測缺少的分類與帳戶，詢問是否自動建立
  - 轉帳配對：轉出與轉入依日期+金額自動配對，建立 linked_id 關聯
  - 分類比對支援「父分類 > 子分類」格式
  - 匯入完成後清空匯入頁面，顯示匯入結果（成功/略過/錯誤筆數）
  - 支援大量資料匯入（JSON body 上限 50MB）

#### FR-072 匯出分類

- **描述：** 使用者可將分類結構匯出為 CSV 檔案
- **格式：** UTF-8 BOM CSV
- **欄位：** 類型、分類名稱、上層分類、顏色
- **處理規則：** 父分類的上層分類欄位為空，子分類填入父分類名稱

#### FR-073 匯入分類

- **描述：** 使用者可匯入 CSV 格式的分類結構
- **處理規則：**
  - 先建立父分類，再建立子分類
  - 已存在的分類自動略過不重複建立
  - 匯入完成後顯示新增/略過數量

#### FR-074 匯出股票紀錄

- **描述：** 使用者可將股票交易紀錄與股利紀錄分別匯出為 CSV 檔案
- **格式：** UTF-8 BOM CSV
- **交易紀錄欄位：** 日期、股票代號、股票名稱、類型（買進/賣出）、股數、成交價、手續費、交易稅、帳戶、備註
- **股利紀錄欄位：** 日期、股票代號、股票名稱、現金股利、股票股利、備註

#### FR-075 匯入股票紀錄

- **描述：** 使用者可匯入 CSV 格式的股票交易紀錄或股利紀錄
- **處理規則：**
  - 第一行為標題列，自動跳過
  - 匯入的股票代號若尚未建立持倉，系統自動新增股票
  - 類型欄位支援「買進」、「賣出」（或英文 `buy`、`sell`）
  - 匯入完成後顯示匯入/略過/錯誤筆數
- **API 端點：**
  - `POST /api/stock-transactions/import`
  - `POST /api/stock-dividends/import`

### 3.10 股票投資管理

#### FR-090 股票持倉管理

- **描述：** 使用者可新增、編輯、刪除股票持倉
- **輸入：** 股票代號、股票名稱、**類型**（一般股票 / ETF / 權證）、目前股價
- **處理規則：**
  - 股票代號不可重複
  - 持有股數、平均成本、損益由系統自動根據交易記錄計算（FIFO 先進先出法）
  - 股票類型（`stock_type`）決定賣出時的證交稅稅率（一般股票 0.3%、ETF/權證 0.1%）
- **顯示內容：**
  - 投資組合總覽卡片：總市值、總成本、總損益、報酬率
  - 個股卡片：股票代號與名稱、持有股數、平均成本、目前股價、市值、損益與報酬率
  - 獲利以綠色顯示（▲），虧損以紅色顯示（▼）

#### FR-091 TWSE 股票代號查詢（三段價格策略）

- **描述：** 新增股票時輸入代號，自動從臺灣證券交易所 OpenAPI 查詢股票名稱與最新股價
- **處理規則：**
  - 後端代理 TWSE API（解決 CORS 限制）；前端防抖（500ms）避免頻繁查詢
  - 前端依台灣時間（UTC+8，瀏覽器時鐘）判斷市場狀態，選擇對應 API：
    1. **盤中**（週一～五 09:00–13:30）：呼叫 `?realtime=1`，後端查 `mis.twse.com.tw` 即時成交價，快取 1 分鐘
    2. **盤後**（週一～五 13:30 後）：呼叫 `?date=YYYYMMDD`，後端查 TWSE `STOCK_DAY` 今日收盤價，快取 5 分鐘
    3. **非交易日 / 盤前**：不帶參數，後端查 `STOCK_DAY_ALL`（T+1 資料），快取 10 分鐘
  - 支援上市（TSE）與上櫃（OTC）股票
  - 查詢成功顯示「✓ 股票名稱 價格類型 $xxx（時間/日期）」綠色提示，自動填入名稱與股價
  - 查詢失敗顯示「找不到此股票代號」紅色提示
- **API 來源：**
  - 即時：`https://mis.twse.com.tw/stock/api/getStockInfo.jsp`
  - 盤後：`https://www.twse.com.tw/exchangeReport/STOCK_DAY`
  - 備援：`https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL`

#### FR-092 股票交易記錄

- **描述：** 使用者可記錄股票的買入與賣出交易
- **輸入：** 股票代號（文字輸入，自動查詢 TWSE）、交易類型（買入/賣出）、日期、股數、每股價格、手續費、交易稅、證券帳戶、備註
- **處理規則：**
  - 股票代號輸入後自動查詢 TWSE 或比對已有持倉，顯示查詢狀態
  - 若輸入的股票代號尚未建立持倉，儲存時自動新增股票（從 TWSE 取得名稱與股價）
  - 手續費預設自動計算：`Math.floor(成交金額 × 0.1425%)`，整股最低 **20 元**
  - 證交稅僅賣出時計算：`Math.floor(成交金額 × 稅率)`，最低 **1 元**
    - 一般股票：0.3%；ETF / 權證：0.1%（依股票的 `stock_type` 決定）
  - 輸入股數或價格時即時計算並顯示費用摘要（成交金額、手續費、交易稅、總成本/淨收入）
  - 交易記錄影響持倉的股數、平均成本與損益計算
  - 買入/賣出按鈕切換時即時變色（買入綠色、賣出紅色）
- **顯示：** 表格列出所有交易記錄，包含日期、股票、類型（買入綠標/賣出紅標）、股數、價格、手續費、稅、總額、備註
- **篩選：** 支援依股票代號或名稱搜尋篩選（下拉式搜尋選單）

#### FR-093 股票股利記錄

- **描述：** 使用者可記錄股票股利（現金股利與股票股利）
- **輸入：** 股票代號（文字輸入，自動查詢 TWSE）、除權息日期、現金股利（元）、股票股利（股）、備註
- **處理規則：**
  - 股票代號輸入後自動查詢 TWSE 或比對已有持倉
  - 若輸入的股票代號尚未建立持倉，儲存時自動新增股票
  - 現金股利與股票股利至少填一項
  - 股票股利增加持有股數
- **顯示：** 表格列出所有股利記錄
- **篩選：** 支援依股票代號或名稱搜尋篩選（下拉式搜尋選單）

#### FR-094 批次更新股價

- **描述：** 使用者可一次更新所有股票的目前股價
- **操作方式：**
  - 開啟股價更新 Modal，顯示所有持股與目前股價輸入框
  - 「從證交所取得最新股價」按鈕：批次從 TWSE OpenAPI 取得所有持股的最新收盤價
  - 手動修改個別股價
  - 確認後批次更新所有股價

#### FR-095 股票損益計算（FIFO）

- **描述：** 系統自動以 FIFO（先進先出）法計算每檔股票的損益
- **計算邏輯：**
  - 買入交易累加至 FIFO 佇列（含手續費分攤至成本）
  - 賣出時依序從最早買入批次扣除，計算已實現損益
  - 未賣出部位以 FIFO 剩餘批次計算持有股數與平均成本
  - 未實現損益 = (目前股價 - 平均成本) × 持有股數
  - 報酬率 = 未實現損益 / 總成本 × 100%

#### FR-096 除權息自動同步

- **描述：** 使用者可一鍵從 TWSE 除權息公告自動同步股利紀錄
- **操作：** 股利紀錄 Tab 中的「同步除權息」按鈕
- **處理規則：**
  - 後端依使用者最早交易日期至今日，按年分段查詢 TWSE `TWT49U` 除權息列表
  - 僅處理使用者持有的股票代號
  - 計算每個除權息日的持股數（依交易紀錄累計，若持股為 0 則跳過）
  - 純現金股利（「息」）直接取 `權值+息值` 為每股金額
  - 含股票股利（「權」、「權息」）查詢 `TWT49UDetail` 取得每股現金股利與每千股配股數
  - 現金股利 = 持股數 × 每股現金股利（四捨五入）
  - 股票股利 = 持股數 × 每千股配股數 / 1000
  - 已存在同日期同股票的股利紀錄不重複新增
  - 備註自動標示「TWSE自動同步（每股$X.XX）」
- **API 來源：**
  - 列表：`https://www.twse.com.tw/rwd/zh/exRight/TWT49U?response=json&startDate=YYYYMMDD&endDate=YYYYMMDD`
  - 明細：`https://www.twse.com.tw/rwd/zh/exRight/TWT49UDetail?response=json&STK_NO=SYMBOL&T1=YYYYMMDD`
- **快取：** 30 分鐘

#### FR-097 實現損益紀錄

- **描述：** 以獨立 Tab 顯示每筆賣出交易的 FIFO 實現損益明細
- **彙總卡片：**
  - 總實現損益（所有已實現賣出的損益合計）
  - 整體報酬率（總實現損益 / 總成本）
  - 今年實現損益（當年度賣出的損益合計）
  - 已實現筆數
- **表格欄位：** 賣出日期、股票（代號+名稱）、賣出股數、賣出均價、成本均價（FIFO）、手續費+稅、實現損益、報酬率
- **篩選：** 支援依股票代號或名稱搜尋篩選（下拉式搜尋選單）
- **計算邏輯：**
  - 對每筆賣出交易，依 FIFO 順序從買入批次中扣除，計算該筆賣出的成本基礎（含手續費按比例分攤）
  - 賣出收入 = 股數 × 賣出價 - 手續費 - 證交稅
  - 實現損益 = 賣出收入 - FIFO 成本基礎
  - 報酬率 = 實現損益 / FIFO 成本基礎 × 100%

```
            ├── 匯出交易記錄（CSV）
            ├── 分類管理（匯出/匯入 CSV）
            ├── 股票紀錄（匯出/匯入 交易紀錄 + 股利紀錄 CSV）
            └── 匯入交易記錄（拖曳上傳、預覽、自動建立）
```

### 6.2 介面設計原則

- 以卡片式佈局呈現資訊
- 一致的色彩系統：收入綠色、支出紅色、轉帳藍色、主色調紫色（#6366f1）
- 金額顯示格式：NT$ 1,234（千分位逗號）
- 日期顯示格式：YYYY-MM-DD
- 重要操作提供 Toast 通知回饋（成功綠色、錯誤紅色）
- 浮動按鈕（FAB）一鍵新增交易記錄
- 側邊欄導航（桌面常駐、手機漢堡選單展開）
