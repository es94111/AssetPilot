## 1. 簡介

### 1.1 目的

本文件定義「資產管理」網頁應用程式的軟體需求規格，作為系統設計、開發與測試的依據。

### 1.2 範圍

本系統為一個網頁應用程式，提供使用者記錄日常收入與支出、管理帳戶轉帳、設定預算、檢視統計報表、管理分類（含子分類）、匯出匯入資料、股票投資組合管理等功能，協助使用者掌握個人財務狀況與投資績效。

### 1.3 名詞定義

| 名詞     | 定義                                       |
| -------- | ------------------------------------------ |
| 交易記錄 | 一筆收入、支出或轉帳的資料                 |
| 分類     | 交易記錄的類別（如餐飲、交通），支援子分類 |
| 子分類   | 隸屬於父分類的細項分類（如餐飲 > 早餐）    |
| 帳戶     | 資金來源（如現金、銀行帳戶、信用卡）       |
| 預算     | 使用者設定的月度支出上限                   |
| 固定收支 | 週期性自動產生的交易記錄                   |
| 轉帳     | 帳戶間的資金移轉（產生轉出+轉入配對記錄）  |
| 批次操作 | 一次對多筆交易進行刪除或變更               |
| 股票持倉 | 使用者持有的股票資產記錄                   |
| 股票交易 | 股票買入或賣出的交易記錄                   |
| 股票股利 | 股票現金股利或股票股利的除息基準記錄       |
| FIFO     | 先進先出法，用於計算股票成本基礎與損益     |
| TWSE     | 臺灣證券交易所（Taiwan Stock Exchange）    |

### 1.4 參考文件

- IEEE 830-1998 軟體需求規格標準

## 2. 整體描述

### 2.1 產品概觀

本系統為獨立的網頁應用程式，使用者透過瀏覽器即可使用。系統採用前後端一體化架構（Node.js + Express + SQLite），支援桌面與行動裝置瀏覽。

### 2.2 技術架構

| 層級     | 技術                                             |
| -------- | ------------------------------------------------ |
| 前端     | 原生 HTML / CSS / JavaScript（單頁應用程式 SPA） |
| 後端     | Node.js + Express                                |
| 資料庫   | SQLite（透過 sql.js）                            |
| 驗證     | JWT（JSON Web Token），有效期 7 天               |
| 密碼     | bcryptjs 加密儲存                                |
| SSO      | Google Identity Services（選配）                 |
| 圖表     | Chart.js                                         |
| 圖示     | Font Awesome 6                                   |
| 外部 API | TWSE OpenAPI、exchangerate-api.com 匯率 API、Google Identity Services、ipinfo.io |

### 2.3 使用者角色

| 角色       | 說明                              |
| ---------- | --------------------------------- |
| 管理員     | 可管理註冊政策與使用者帳號        |
| 一般使用者 | 註冊帳號後使用所有功能            |
| 訪客       | 未登入狀態，僅可瀏覽登入/註冊頁面 |

### 2.4 操作環境

- **前端：** 現代瀏覽器（Chrome、Firefox、Safari、Edge 最新版本）
- **後端：** Node.js 18+
- **部署：** 本地伺服器或雲端，預設 port 3000

### 2.5 限制條件

- 系統僅支援新台幣（NT$）為預設貨幣
- 不提供銀行帳戶自動連結功能
- 單一使用者帳號不支援多人共用

## 3. 功能需求

### 3.1 使用者管理

#### FR-001 使用者註冊

- **描述：** 使用者可透過電子郵件與密碼建立帳號
- **輸入：** 電子郵件、密碼、顯示名稱
- **處理規則：**
  - 電子郵件格式驗證，不可重複註冊
  - 密碼長度至少 6 字元
- **輸出：** 註冊成功後自動登入，建立預設分類（含子分類）與預設帳戶

#### FR-002 使用者登入 / 登出

- **描述：** 使用者輸入帳號密碼登入系統
- **輸入：** 電子郵件、密碼
- **處理規則：**
  - 驗證帳號密碼是否正確
  - 登入成功後發放 JWT Token，有效期 7 天
  - 登入成功時記錄登入稽核（登入時間、IP 位址、登入方式、身份）
  - 登入 API 回傳 `currentLogin`，確保前端可立即顯示本次登入紀錄
  - Token 儲存於 localStorage，重新開啟瀏覽器自動恢復登入
- **輸出：** 登入成功導向主頁面（依 URL 路由）；登出後導向登入頁

#### FR-003 Google SSO 登入（選配）

- **描述：** 使用者可使用 Google 帳號快速登入或註冊
- **前置條件：** 伺服器已設定 `GOOGLE_CLIENT_ID` 與 `GOOGLE_CLIENT_SECRET` 環境變數
- **處理規則：**
  - 前端透過 `/api/config` 取得 Google Client ID，動態顯示 Google 登入按鈕
  - 使用者點擊 Google 按鈕，透過 Google Identity Services 啟動 OAuth Authorization Code Flow
  - 前端先向後端取得一次性 state，隨授權請求送出並在回呼時比對
  - 後端僅接受具有效 state 的授權碼，並向 Google token endpoint 交換 token 後取得使用者資訊
  - 若該 Email 已存在帳號，直接登入；若不存在，自動建立帳號（含預設分類與帳戶）
  - Google SSO 建立的帳號密碼欄位設為隨機 hash（不可用密碼登入，僅能用 Google）
- **輸出：** 登入成功導向主頁面，與一般登入相同
- **備註：** 未設定 Client ID 時，Google 按鈕自動隱藏，不影響帳號密碼登入功能

#### FR-006 管理員身分初始化

- **描述：** 系統第一個建立的使用者自動成為管理員
- **處理規則：**
  - 當資料庫尚無使用者時，第一筆註冊帳號（一般註冊或 Google 首次註冊）需標記為管理員
  - 舊資料升級時若尚無任何管理員，需將最早建立的使用者升級為管理員

#### FR-007 註冊政策管理

- **描述：** 管理員可決定是否開放公開註冊，並可設定可註冊 Email 白名單
- **處理規則：**
  - 可切換公開註冊開關（開啟/關閉）
  - 可維護 Email 白名單（每行一個 Email）
  - 若白名單非空，僅白名單 Email 可註冊（一般註冊與 Google 首次註冊皆套用）
  - 若白名單為空且公開註冊關閉，訪客不可自行註冊

#### FR-008 管理員建立與刪除使用者

- **描述：** 管理員可直接建立帳號與刪除指定使用者
- **處理規則：**
  - 建立帳號時可指定是否授與管理員身分
  - 刪除使用者時需一併刪除其所有關聯資料（交易、帳戶、分類、預算、股票資料等）
  - 管理員不可透過管理介面刪除自己

#### FR-009 最後管理員保護

- **描述：** 系統需避免管理員帳號全數消失
- **處理規則：**
  - 當系統僅剩一位管理員時，不可刪除此管理員帳號
  - 使用者自行刪除帳號流程亦需套用同一限制

#### FR-032 使用者登入稽核紀錄

- **描述：** 一般使用者可在帳號設定查看自己的登入時間與 IP 紀錄
- **處理規則：**
  - 密碼登入與 Google SSO 登入成功後皆需寫入稽核紀錄
  - 稽核欄位至少包含：登入時間、IP 位址、登入方式、是否以管理員身份登入
    - 稽核列表需顯示 IP 對應國家（以 ipinfo.io 查詢，內網位址顯示 LOCAL）
  - 使用者僅可查詢自己的登入紀錄（最近 100 筆）

#### FR-033 管理員登入與全站登入稽核查詢

- **描述：** 管理員可查看自身管理員登入紀錄與全部使用者登入紀錄
- **處理規則：**
  - 管理員可查詢自己的管理員身份登入紀錄（最近 200 筆）
  - 管理員可查詢全站使用者登入紀錄（最近 500 筆）
  - 全站登入紀錄必須包含失敗登入嘗試（如帳號不存在、密碼錯誤、缺少憑證、暫時鎖定）
  - 全站登入紀錄需顯示成功/失敗結果與失敗原因
  - 管理員登入紀錄與全站登入紀錄需顯示 IP 對應國家代碼
  - 管理員登入紀錄需支援單筆刪除與多選批次刪除
  - 管理員登入紀錄需支援手動同步，且顯示上次同步時間
  - 全站使用者登入紀錄需支援手動同步，且顯示上次同步時間
  - 全站使用者登入紀錄需支援單筆刪除與多選批次刪除
  - 當舊資料缺少主鍵時，仍需可透過備援識別（例如時間戳）執行單筆刪除
  - 非管理員不可存取管理員登入稽核查詢 API

#### FR-034 修改顯示名稱

- **描述：** 使用者可在帳號設定修改自己的顯示名稱
- **輸入：** 顯示名稱（1 ~ 50 字）
- **處理規則：**
  - 顯示名稱不可空白
  - 儲存成功後，帳號資訊與側邊欄名稱需即時更新
- **輸出：** 顯示名稱更新成功提示

#### FR-004 股票交易紀錄批次刪除與分頁

- **描述：** 股票交易紀錄支援多選批次刪除與分頁瀏覽
- **處理規則：**
  - 每列前方有 checkbox，表頭有全選 checkbox
  - 勾選後顯示批次操作列（已選 N 筆 + 批次刪除按鈕 + 取消按鈕）
  - 每頁筆數可選 10 / 20 / 50 / 100 / 自訂
  - 伺服器端分頁（`page` + `pageSize` 查詢參數）
- **API：**
  - `GET /api/stock-transactions?page=1&pageSize=20` — 回傳 `{ data, total, page, totalPages }`
  - `POST /api/stock-transactions/batch-delete` — 批次刪除（`{ ids: [...] }`）

#### FR-005 股利紀錄批次刪除與分頁

- **描述：** 股利紀錄支援多選批次刪除與分頁瀏覽
- **處理規則：** 同 FR-004
- **API：**
  - `GET /api/stock-dividends?page=1&pageSize=20` — 回傳 `{ data, total, page, totalPages }`
  - `POST /api/stock-dividends/batch-delete` — 批次刪除（`{ ids: [...] }`）

### 3.2 交易記錄管理

#### FR-010 新增交易記錄

- **描述：** 使用者可新增一筆收入或支出記錄
- **輸入：**
  - 類型（收入 / 支出）— 必填
  - 金額 — 必填，正數
  - 日期 — 必填，預設為今天
  - 分類 — 必填（支援子分類，以 optgroup 群組顯示）
  - 帳戶 — 必填
  - 備註 — 選填，最多 200 字
- **處理規則：**
  - 金額不可為 0 或負數
  - 日期可選擇未來日期，供預先規劃交易
- **輸出：** 新增成功 Toast 提示，更新交易列表

#### FR-011 編輯交易記錄

- **描述：** 使用者可修改已存在的交易記錄
- **輸入：** 同 FR-010
- **處理規則：** 同 FR-010，記錄最後修改時間
- **輸出：** 修改成功 Toast 提示

#### FR-012 刪除交易記錄

- **描述：** 使用者可刪除交易記錄
- **處理規則：**
  - 刪除前需二次確認（確認 Modal）
  - 若為轉帳交易，同時刪除關聯的 linked 交易
- **輸出：** 刪除成功 Toast 提示，更新交易列表

#### FR-013 檢視交易記錄列表

- **描述：** 使用者可瀏覽交易記錄列表
- **顯示欄位：** 勾選框、日期、類型、分類（含父分類 > 子分類顯示）、金額、帳戶、備註、操作
- **處理規則：**
  - 依日期由新到舊排序
  - 支援分頁，每頁筆數可選：10 / 20 / 50 / 100 / 自訂
  - 類型以色彩標籤顯示（支出紅、收入綠、轉帳藍）
  - 未來日期交易需顯示「未來」標籤，與已發生交易區分

#### FR-014 搜尋與篩選交易記錄

- **描述：** 使用者可依條件篩選交易記錄
- **篩選條件：**
  - 日期範圍（起始日 ~ 結束日）
  - 類型（收入 / 支出 / 轉帳 / 未來交易 / 全部）
  - 分類（以 optgroup 群組顯示子分類）
  - 帳戶
  - 關鍵字搜尋（備註欄位）
  - 每頁筆數（10 / 20 / 50 / 100 / 自訂輸入）
- **操作：** 篩選按鈕套用、重置按鈕恢復預設

#### FR-015 批次操作

- **描述：** 使用者可多選交易記錄進行批次操作
- **功能：**
  - 勾選框：每列左側 checkbox，表頭有全選 checkbox（支援全選/半選/取消）
  - 批次操作列：勾選後顯示紫色操作列，顯示已選筆數
  - **批次刪除：** 一次刪除多筆交易（含轉帳關聯刪除），需二次確認
  - **批次變更分類：** 一次將多筆交易變更為指定分類
  - **批次變更帳戶：** 一次將多筆交易變更為指定帳戶
  - **批次變更日期：** 一次將多筆交易變更為指定日期
  - **取消選取：** 清除所有勾選，隱藏操作列

### 3.3 帳戶間轉帳

#### FR-016 新增轉帳記錄

- **描述：** 使用者可記錄帳戶間的資金移轉（亦可用於信用卡繳款）
- **輸入：** 轉出帳戶、轉入帳戶、金額、日期、備註
- **處理規則：**
  - 自動產生一對 transfer_out 和 transfer_in 交易記錄
  - 兩筆記錄透過 linked_id 互相關聯
  - 刪除其中一筆時同步刪除另一筆
  - **應用情境（信用卡繳款）：** 信用卡消費時記為「支出」，該帳戶呈現負額；繳款時記為「轉帳（銀行 -> 信用卡）」，轉入金額用以中和負額，避免重複認列支出。

### 3.4 分類管理

#### FR-020 檢視分類列表

- **描述：** 使用者可檢視所有收支分類，以階層式結構顯示
- **預設支出分類及子分類：**
  - 餐飲：早餐、午餐、晚餐、飲料、點心
  - 交通：公車/捷運、計程車/Uber、加油、停車費、高鐵/火車
  - 購物：日用品、服飾、3C產品、家電
  - 娛樂：電影/影音、遊戲、旅遊、運動健身
  - 居住：房租/房貸、水電費、網路費、管理費
  - 醫療：掛號費、藥品、保健食品
  - 教育：學費、書籍、線上課程
  - 其他
- **預設收入分類：** 薪資、獎金、投資、兼職、其他
- **顯示方式：**
  - 父分類顯示完整寬度，帶 `+`（新增子分類）和編輯按鈕
  - 子分類以網格佈局縮排顯示，帶左側藍色邊框和箭頭圖示
  - 非預設分類可刪除

#### FR-021 新增分類

- **描述：** 使用者可新增頂層分類或子分類
- **輸入：** 分類名稱、類型（收入/支出）、顏色、上層分類（可選）
- **處理規則：**
  - 同父分類下名稱不可重複
  - 子分類的類型必須與父分類相同
  - 子分類僅支援一層（不可建立孫分類）

#### FR-022 編輯分類

- **描述：** 使用者可修改分類名稱與顏色
- **處理規則：**
  - 若為父分類且有子分類，不可變更為子分類
  - 預設分類不可刪除

#### FR-023 刪除分類

- **描述：** 使用者可刪除非預設分類
- **處理規則：**
  - 該分類下若有交易記錄，不可刪除
  - 刪除父分類時，若子分類下有交易記錄，不可刪除
  - 刪除父分類時連帶刪除所有子分類

#### FR-024 預設子分類自動建立

- **描述：** 新使用者註冊時自動建立完整的預設分類與子分類
- **處理規則：** 既有使用者在系統升級後，自動補建缺少的預設子分類

### 3.5 帳戶管理

#### FR-030 新增帳戶

- **描述：** 使用者可新增資金帳戶
- **輸入：** 帳戶名稱、初始餘額、圖示
- **預設帳戶：** 現金

#### FR-031 編輯帳戶

- **描述：** 使用者可修改帳戶名稱、圖示

#### FR-032 刪除帳戶

- **描述：** 使用者可刪除帳戶
- **處理規則：** 帳戶下若有交易記錄，不可刪除

#### FR-033 檢視帳戶餘額

- **描述：** 顯示各帳戶目前餘額與總資產
- **計算方式：** 初始餘額 + 收入總額 - 支出總額 + 轉入金額 - 轉出金額

### 3.6 預算管理

#### FR-040 設定月度預算

- **描述：** 使用者可設定每月總支出預算或分類預算
- **輸入：** 預算金額、適用月份、分類（可選，不選為總預算）
- **分類選擇：** 支援子分類群組顯示

#### FR-041 預算進度顯示

- **描述：** 儀表板顯示當月預算使用進度
- **顯示內容：**
  - 已使用金額 / 預算金額
  - 使用百分比進度條
  - 剩餘可用金額

#### FR-091 TWSE 股票代號查詢（三段價格策略）

- **描述：** 新增股票時輸入代號，自動從臺灣證券交易所 OpenAPI 查詢股票名稱與最新股價
- **處理規則：**
  - 後端代理 TWSE API（解決 CORS 限制）；前端防抖（500ms）避免頻繁查詢
  - 前端依台灣時間（UTC+8，瀏覽器時鐘）判斷市場狀態，選擇對應 API：
    1. **盤中**（週一～五 09:00–13:30）：呼叫 `?realtime=1`，後端查 `mis.twse.com.tw` 即時成交價，快取 1 分鐘
    2. **盤後**（週一～五 13:30 後）：呼叫 `?date=YYYYMMDD`，後端查 TWSE `STOCK_DAY` 今日收盤價，快取 5 分鐘
    3. **非交易日 / 盤前**：不帶參數，後端查 `STOCK_DAY_ALL`（T+1 資料），快取 10 分鐘
  - 支援上市（TSE）與上櫃（OTC）股票
  - 查詢成功顯示「✓ 股票名稱 價格類型 $xxx（時間/日期）」綠色提示，自動填入名稱與股價
  - 查詢失敗顯示「找不到此股票代號」紅色提示
- **API 來源：**
  - 即時：`https://mis.twse.com.tw/stock/api/getStockInfo.jsp`
  - 盤後：`https://www.twse.com.tw/exchangeReport/STOCK_DAY`
  - 備援：`https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL`

#### FR-092 股票交易記錄

- **描述：** 使用者可記錄股票的買入與賣出交易
- **輸入：** 股票代號（文字輸入，自動查詢 TWSE）、交易類型（買入/賣出）、日期、股數、每股價格、手續費、交易稅、證券帳戶、備註
- **處理規則：**
  - 股票代號輸入後自動查詢 TWSE 或比對已有持倉，顯示查詢狀態
  - 若輸入的股票代號尚未建立持倉，儲存時自動新增股票（從 TWSE 取得名稱與股價）

  - 證交稅僅賣出時計算：`Math.floor(成交金額 × 稅率)`，最低 **1 元**
    - 一般股票：0.3%；ETF / 權證：0.1%（依股票的 `stock_type` 決定）
  - 輸入股數或價格時即時計算並顯示費用摘要（成交金額、手續費、交易稅、總成本/淨收入）
  - 交易記錄影響持倉的股數、平均成本與損益計算
  - 買入/賣出按鈕切換時即時變色（買入綠色、賣出紅色）
- **顯示：** 表格列出所有交易記錄，包含日期、股票、類型（買入綠標/賣出紅標）、股數、價格、手續費、稅、總額、備註
- **篩選：** 支援依股票代號或名稱搜尋篩選（下拉式搜尋選單）

#### FR-093 股票股利記錄

- **描述：** 使用者可記錄股票股利（現金股利與股票股利）
- **輸入：** 股票代號（文字輸入，自動查詢 TWSE）、除息日期、現金股利（元）、股票股利（股）、備註

- **處理規則：**
  - 股票代號輸入後自動查詢 TWSE 或比對已有持倉

- **操作方式：**

  - 手動修改個別股價
  - 確認後批次更新所有股價

#### FR-095 股票損益計算（FIFO）

- **描述：** 系統自動以 FIFO（先進先出）法計算每檔股票的損益
- **計算邏輯：**
  - 買入交易累加至 FIFO 佇列（含手續費分攤至成本）
  - 賣出時依序從最早買入批次扣除，計算已實現損益
  - 未賣出部位以 FIFO 剩餘批次計算持有股數與平均成本
  - 未實現損益 = (目前股價 - 平均成本) × 持有股數
  - 報酬率 = 未實現損益 / 總成本 × 100%

#### FR-096 除權息自動同步

- **描述：** 使用者可一鍵從 TWSE 除權息公告自動同步股利紀錄
- **操作：** 股利紀錄 Tab 中的「同步除權息」按鈕
- **處理規則：**
  - 後端依使用者最早交易日期至今日，按年分段查詢 TWSE `TWT49U` 除權息列表
  - 僅處理使用者持有的股票代號
  - 計算每個除權息日的持股數（依交易紀錄累計，若持股為 0 則跳過）
  - 純現金股利（「息」）直接取 `權值+息值` 為每股金額
  - 含股票股利（「權」、「權息」）查詢 `TWT49UDetail` 取得每股現金股利與每千股配股數
  - 現金股利 = 持股數 × 每股現金股利（四捨五入）
  - 股票股利 = 持股數 × 每千股配股數 / 1000
  - 已存在同日期同股票的股利紀錄不重複新增
  - 備註自動標示「TWSE自動同步（每股$X.XX）」
- **API 來源：**
  - 列表：`https://www.twse.com.tw/rwd/zh/exRight/TWT49U?response=json&startDate=YYYYMMDD&endDate=YYYYMMDD`
  - 明細：`https://www.twse.com.tw/rwd/zh/exRight/TWT49UDetail?response=json&STK_NO=SYMBOL&T1=YYYYMMDD`
- **快取：** 30 分鐘

#### FR-097 實現損益紀錄

- **描述：** 以獨立 Tab 顯示每筆賣出交易的 FIFO 實現損益明細
- **彙總卡片：**
  - 總實現損益（所有已實現賣出的損益合計）
  - 整體報酬率（總實現損益 / 總成本）
  - 今年實現損益（當年度賣出的損益合計）
  - 已實現筆數
- **表格欄位：** 賣出日期、股票（代號+名稱）、賣出股數、賣出均價、成本均價（FIFO）、手續費+稅、實現損益、報酬率
- **篩選：** 支援依股票代號或名稱搜尋篩選（下拉式搜尋選單）
- **計算邏輯：**
  - 對每筆賣出交易，依 FIFO 順序從買入批次中扣除，計算該筆賣出的成本基礎（含手續費按比例分攤）
  - 賣出收入 = 股數 × 賣出價 - 手續費 - 證交稅
  - 實現損益 = 賣出收入 - FIFO 成本基礎
  - 報酬率 = 實現損益 / FIFO 成本基礎 × 100%

#### FR-098 股票定期定額

- **描述：** 使用者可設定股票定期定額，系統依週期自動產生買進交易
- **設定欄位：** 股票、每期預算、週期、起始日、證券帳戶、備註、啟用/停用
- **處理規則：**
  - 可新增、編輯、刪除與啟用/停用定期定額設定
  - 系統於登入時自動檢查並補齊到期排程
  - 每次執行以「每期預算 ÷ 當前股價」計算可買股數（無法買 1 股時略過）
  - 自動產生的買進交易會附註記，便於後續辨識

  - 位於收支管理頁（`transactions` / `reports` / `budget` / `accounts` / `categories` / `recurring`）時顯示「新增交易」
  - 位於股票紀錄頁（`stocks` 含子分頁）時顯示「新增股票交易紀錄」
  - 其他頁面（如儀表板、設定、API 使用與授權）不顯示快速新增按鈕
- **操作規則：**
  - 點擊「新增交易」開啟交易新增 Modal
  - 點擊「新增股票交易紀錄」開啟股票交易新增 Modal

## 4. 非功能需求

### 4.1 效能需求

| 項目         | 要求                         |
| ------------ | ---------------------------- |
| 頁面載入時間 | 首頁 ≤ 3 秒（一般網路環境） |
| API 回應時間 | 單筆 CRUD 操作 ≤ 500ms      |
| 大量匯入     | 支援 9000+ 筆一次匯入        |
| Request Body | JSON 上限 50MB               |

### 4.2 安全性需求

- 密碼使用 bcryptjs 加密儲存
- JWT Token 用於身份驗證，有效期 7 天
- 所有 API 端點（除 auth 相關）透過 authMiddleware 保護
- 系統更新 API 僅允許管理員呼叫
- 輸入資料進行 HTML 跳脫防護（XSS）
- 啟用 Content Security Policy（CSP）限制腳本與外部資源來源
- 靜態檔僅允許白名單公開資源，避免專案根目錄敏感檔案外洩

### 4.3 可用性需求

- 響應式設計（RWD），支援手機、平板、桌面
- 支援繁體中文介面
- 操作流程直覺化，新增一筆記錄不超過 3 個步驟
- 重要操作提供 Toast 通知回饋
- 刪除操作需二次確認（確認 Modal）

### 4.4 可靠性需求

- SQLite 資料庫即時儲存（每次寫入後 saveDB）
- 異常情況提供友善錯誤 Toast 訊息
