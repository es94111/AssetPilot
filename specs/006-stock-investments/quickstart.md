# Quickstart 驗證劇本：股票投資（Stock Investments）

**Branch**: `006-stock-investments` | **Date**: 2026-04-26 | **Plan**: [plan.md](./plan.md)

本檔提供本功能的人工驗證劇本。**無自動化測試框架**（與 001 ~ 005 一致）；驗證方式為 DevTools Network 面板 + 瀏覽器手動操作。劇本依 spec 7 個 user story 排列，並涵蓋 19 條 clarification 的關鍵驗收路徑。

---

## 前置作業

1. 確保本地 `database.db` 為乾淨狀態（或測試帳號）。
2. 啟動伺服器：`npm start`（既有指令，無變更）。
3. 開啟 Chrome / Edge → 登入測試帳號。
4. 預設 `TWSE_MAX_CONCURRENCY` env var 為 5；如需測試低並發可改為 `TWSE_MAX_CONCURRENCY=1` 重啟。

## US1 — 開啟股票頁立即掌握目前持倉與報酬率（P1）

### 步驟

1. 預先建立 3 檔持股的買入紀錄（如 2330 / 0050 / 6488，各 1000 股）。
2. 點主選單「股票」開啟股票頁。

### 預期觀察

- ✅ 投資組合總覽卡顯示 4 個彙整數字：總市值、總成本、總損益（含 ▲／▼）、整體報酬率。
- ✅ 整體報酬率為**金額加權**：手動計算 `(總市值 − 總成本) / 總成本 × 100%` 與顯示一致（**Pass 2 Q4**）。
- ✅ 3 張個股卡片，每張卡顯示 8 個欄位（代號 / 名稱 / 股數 / 平均成本 / 目前股價 / 市值 / 損益 / 報酬率）。
- ✅ 各卡片顯示「最後查價時間」與資料來源（即時／收盤／T+1）標示（**Pass 1 Q5**）。
- ✅ 損益正值綠色 ▲、負值紅色 ▼、**剛好為 0 灰色（無符號）**（**Pass 4 Q4**）。
- ✅ 首次載入無 `current_price` 的個股市值欄位顯示「—」（FR-005 首次新增 fallback）。

### 邊界驗證

- 賣光某檔持股至股數 = 0 → 該檔不出現於個股卡片列表，但實現損益 Tab 仍可看見其賣出歷史。
- 將某檔股票 `updated_at` 手動改為 25 小時前（直接改 DB）→ 重新整理 → 該卡套用 ⚠ 警示色（如橘色或黃色），但市值仍以該過期價格計算（**不**回退為「—」）（**Pass 1 Q5 第二段**）。

## US2 — 紀錄一筆股票買賣並即時看到費用試算（P1）

### 步驟

1. 點「新增交易」開啟 Modal。
2. 選「買入」 → 輸入代號 `2330` → 等待 500ms。

### 預期觀察

- ✅ 輸入欄旁顯示「✓ 台積電 即時 $XXX（HH:MM）」綠色提示（FR-007）。
- ✅ Modal 顯示自動判定的 `stockType = 'stock'`（一般）下拉選單，可手動覆寫（**Pass 3 Q1**）。

### 子步驟 — 費用試算

3. 輸入股數 `1000`、每股 `600`。
4. 觀察 Modal 下方費用摘要區塊。

### 預期觀察

- ✅ 「成交金額 600,000」「手續費 855」（= ⌊600,000 × 0.1425%⌋ = 855，整股最低 20 元保底）「交易稅 0」「總成本 600,855」即時更新（< 200ms，**SC-002**）。
- ✅ 切到「賣出」 → 手續費 855 不變、證交稅 1,800（= ⌊600,000 × 0.3%⌋）、淨收入 597,345。
- ✅ 切換按鈕配色即時變化（買入綠 / 賣出紅）（FR-009）。

### 邊界驗證

- 輸入股數 `1.5`（小數）→ 試圖送出 → 系統 reject「股數必須為整數」（FR-010）。
- 輸入無效代號 `9999` → 等待 500ms → 紅色提示「找不到此股票代號」（FR-007 + Pass 3 Q4）。
- 試輸入代號含中文「2330台積」→ 前端正則阻擋（不送請求；**Pass 3 Q4**）。
- 輸入「買入 1 股 × $50」（零股）→ 手續費 1 元（零股最低，非整股 20 元，**Pass 2 Q5 + FR-011**）。
- 選 ETF（如 0050）並輸入「賣出 1000 × $130」→ 證交稅 130（0.1%，非 0.3%，FR-012）；**stockType 自動判定為 'etf'**（**Pass 3 Q1**）。

### 子步驟 — 賣出鏈式約束

5. 持股 0050 共 500 股；試圖補錄一筆「2025-08-10 賣出 600 股」。
6. 送出。

### 預期觀察

- ✅ 系統 reject「賣出股數不可超過 2025-08-10 當下持有 (500 股)」（**Pass 3 Q2**）。
- ✅ 改為賣 300 股，但若該補錄會造成 2025-09-15 持有量為負（後續又有賣出），系統 reject 並指出衝突日期。

## US3 — 記錄股利並支援自動同步除權息（P1）

### 子步驟 — 純股票股利紀錄

1. 持股 2330 共 1000 股、新增股利「除息日 2025-09-12 / 現金 0 / 股票股利每千股 100 股」。
2. 不選入款帳戶（試 leave it blank）。
3. 送出。

### 預期觀察

- ✅ 系統接受（**Pass 2 Q2** — 純股票股利 accountId 選填）。
- ✅ 寫入後 2330 持有股數變為 1100（baseline 100 股股票股利配發）。
- ✅ DB 內可見 `stock_transactions` 多一筆 `(price=0, shares=100, type='buy', note LIKE '%股票股利配發%')` 合成交易（**Pass 1 Q1 + Pass 2 Q3 § 3**）。
- ✅ 任何帳戶餘額**未變動**（純股票股利不寫帳戶）。

### 子步驟 — 含現金 + 股票股利

4. 新增股利「2330 / 除息日 2025-08-15 / 現金 4 / 股票股利 0 / 入款帳戶 = X」。
5. 送出。

### 預期觀察

- ✅ 帳戶 X 餘額 +4,000（= 1000 × 4）。
- ✅ 持股股數不變（無股票股利）。

### 子步驟 — 同步除權息阻擋式 Modal

6. 點股利紀錄頁的「同步除權息」按鈕。

### 預期觀察

- ✅ 跳出阻擋式 Modal，顯示進度條 + 當前處理階段（如「正在同步 2025 年（3/5 檔）」）（**Pass 2 Q1**）。
- ✅ 同步期間無法關閉 Modal 或離開股票頁。
- ✅ Modal 提供「取消」按鈕；按下後跳出迴圈、Modal 關閉、toast 顯示部份完成數字。
- ✅ DevTools Network 觀察：每年份一個 `POST /api/stock-dividends/sync?year=YYYY` 請求；逐年完成後更新進度。
- ✅ 完成後 Modal 自動關閉、toast 顯示「同步完成（新增 N / 跳過 M / 失敗 K）」總數。

### 子步驟 — 連動刪除驗證

7. 找到 step 1 寫入的「2330 / 2025-09-12」純股票股利紀錄 → 點刪除。

### 預期觀察

- ✅ 刪除後持有股數從 1100 退回 1000（合成 $0 交易已連動刪除）（**Pass 2 Q3**）。
- ✅ DB 內該筆 `(price=0, shares=100, note LIKE '%股票股利配發%')` transaction 已不存在。
- ✅ 無相關帳戶餘額異動（純股票股利原本就沒寫帳戶）。

## US4 — 設定定期定額並由系統自動於登入時產生交易（P2）

### 子步驟 — 排程建立

1. 點「定期定額」Tab → 新增排程「2330 / 每月 / 5,000 / 起始日 2026-01-01 / 啟用 / 帳戶 = Y」。

### 子步驟 — 登入時自動觸發

2. 模擬「使用者長時間未登入」場景：將 `stock_recurring.last_generated` 設為 NULL（清空，模擬從 start_date 開始）。
3. 直接登出再登入。

### 預期觀察

- ✅ 登入後股票頁的 stock_transactions 多了從 2026-01-01 至今日的所有應觸發期數的交易（**Pass 3 Q3** 全補）。
- ✅ DevTools Network 看到 `processStockRecurring(userId)` 在登入後 3 秒內背景執行（不阻擋登入回應，**SC-005**）。
- ✅ 每筆 INSERT 的 stock_transactions 帶 `recurring_plan_id` + `period_start_date` 兩欄；DB 對應 partial unique index 存在（**Pass 3 Q5**）。
- ✅ 每期使用該期應觸發日（含順延後）的歷史股價（從 TWSE STOCK_DAY 查詢）；非當下即時價（**Pass 3 Q3 後半段**）。

### 子步驟 — 多裝置 race 驗證

4. 同時開兩個瀏覽器（Chrome + Firefox），同一帳號，幾乎同時登入。
5. 觀察 stock_transactions 是否有重複扣款。

### 預期觀察

- ✅ 同期排程僅產生一筆交易（後到的 session INSERT OR IGNORE 安靜跳過，**Pass 3 Q5**）。

### 子步驟 — 預算不足 / 順延驗證

6. 排程 `2330 / 每月 / 1,000`，當前股價 600 → 預期跳過該次（< 1 股）（FR-021）。
7. 排程日 2026-03-01 為週日 → 預期實際交易日寫為 2026-03-02 (週一)（FR-022）。

## US5 — 檢視 FIFO 實現損益（P2）

### 步驟

1. 預先建立交易：「2025-01-15 買 500 × $580」「2025-03-20 買 500 × $620」「2025-08-10 賣 700 × $700」。
2. 點「實現損益」Tab。

### 預期觀察

- ✅ 表格列出 2025-08-10 賣出該筆。
- ✅ 成本均價 ≈ 591（= (500×580 + 200×620) / 700）；賣出均價 700；實現損益 ≈ +72,832；報酬率 ≈ +17.6%（綠色 ▲）。
- ✅ 同一檔的「未賣出剩餘 300 股」於持倉頁的平均成本 = 620（**非**全部買入加權 600；**FR-031**）。
- ✅ 彙總卡 4 數字：總實現損益 / 整體報酬率（金額加權，**Pass 2 Q4**）/ 今年實現損益 / 已實現筆數。

### 邊界驗證

- 僅有買入未曾賣出 → 彙總卡顯示「總實現損益 0、報酬率「—」、已實現筆數 0」；表格區呈現空狀態文案。
- FIFO 計算精度：DevTools Network 觀察 `/api/stock-realized-pl` response，手算與顯示金額誤差 ≤ 1 元（**SC-004 + Pass 4 Q1**）。

## US6 — 一鍵批次更新所有持股的最新股價（P3）

### 步驟

1. 持有 5 檔股票 → 點「批次更新股價」按鈕。
2. 點「從證交所取得最新股價」按鈕。

### 預期觀察

- ✅ Modal 列出 5 列輸入框（預填當前 `current_price`）。
- ✅ 每列右側顯示價格來源（即時 / 收盤 / T+1）與取得時間（FR-034）。
- ✅ DevTools Network 觀察：5 個並發請求；若改 `TWSE_MAX_CONCURRENCY=2` env var 重啟，則改為 3 批 + 2 批分批呼叫（**Pass 4 Q3**）。
- ✅ 模擬 TWSE 失敗（如停掉內網）：失敗的個別股票顯示「查詢失敗」並保留原值；其他股票繼續處理（FR-034）。

### 子步驟 — 標記下市

3. 將某檔（如 6488）的「標為已下市」checkbox 勾選 → 點「確認」。

### 預期觀察

- ✅ DB 內 `stocks.delisted = 1`、`current_price` 凍結為當下值（**Pass 1 Q2 + FR-035a**）。
- ✅ 個股卡片顯示「（已下市）」紅色 badge。
- ✅ 重新點「從證交所取得最新股價」 → 6488 該行**不**送 TWSE 請求（DevTools Network 不見對應請求）。
- ✅ 取消勾選 → `delisted = 0`，恢復 TWSE 查價。

### 子步驟 — 手動覆寫

4. 將某檔股價手動改為與 TWSE 帶回不同的值（如 $610 vs TWSE 帶回 $602） → 確認。

### 預期觀察

- ✅ 該檔 `current_price` 寫入手動值 610，**不**被自動帶入值蓋掉（FR-035）。

## US7（隱含於 Edge Cases） — 權證到期手動處理

### 步驟

1. 持有某權證 X（如 04XXXW）共 1000 股。
2. 權證到期日當天手動建立「賣出 1000 股 × $0」交易。

### 預期觀察

- ✅ 系統接受（**Pass 1 Q4** — 不引入到期日欄位、不自動失效）。
- ✅ 證交稅依 FR-012 = `max(⌊0 × 0.1%⌋, 1)` = 1 元。
- ✅ FIFO 計算實現損益為全額虧損。

## 故障排除

- **登入後排程未觸發**：檢查 server.js login handler 內 `setImmediate(() => processStockRecurring(...))` 是否被 try/catch 吞掉錯誤；console log 應有 `processStockRecurring` 進入點。
- **同步除權息 Modal 卡住**：檢查 `/api/stock-dividends/sync?year=YYYY` 是否 200；前端逐年迴圈是否正確 await。
- **下市股票仍被查價**：檢查 `WHERE delisted = 0` 是否套到所有 TWSE 入口（`updateUserStockPrices()` / `batch-price` / `processStockRecurring()`）。
- **重複扣款**：檢查 `stock_transactions` 的 partial unique index 是否成功建立（`SELECT name FROM sqlite_master WHERE type='index' AND name='idx_stock_tx_recurring_idem'`）。
