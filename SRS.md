## 1. 簡介

### 1.1 目的

本文件定義 **VitaTrack** 健康管理系統的軟體需求，涵蓋身體組成紀錄、飲食紀錄、運動紀錄、健康指標追蹤、目標設定與數據分析等功能模組，作為開發、測試與驗收的依據。

### 1.2 範圍

VitaTrack 是一套個人健康管理應用程式，協助使用者：

- 記錄並追蹤身體組成變化（體重、體脂率、肌肉量等）
- 記錄每日飲食攝取與營養素分析
- 記錄運動與活動量
- 設定健康目標並追蹤達成進度
- 透過圖表與報表分析長期健康趨勢

### 1.3 術語定義

| 術語 | 說明 |
|------|------|
| BMI | 身體質量指數（Body Mass Index） |
| BMR | 基礎代謝率（Basal Metabolic Rate） |
| TDEE | 每日總能量消耗（Total Daily Energy Expenditure） |
| 巨量營養素 | 碳水化合物、蛋白質、脂肪 |
| 微量營養素 | 維生素、礦物質等 |

### 1.4 參考文件

- 衛福部國民健康署「國人膳食營養素參考攝取量」
- WHO BMI 分類標準

## 2. 整體描述

### 2.1 產品願景

提供直覺、簡潔的健康管理工具，讓使用者輕鬆養成記錄習慣，透過數據洞察做出更好的健康決策。

### 2.2 使用者角色

| 角色 | 說明 |
|------|------|
| 一般使用者 | 記錄個人健康數據、查看分析報表 |
| 管理員 | 管理食物資料庫、系統設定 |

### 2.3 運作環境

- **前端**：Web 應用程式（RWD 響應式設計，支援桌面與行動裝置）
- **後端**：RESTful API 服務
- **資料庫**：關聯式資料庫
- **部署**：支援容器化部署（Docker）

### 2.4 設計約束

- 所有日期時間以使用者所在時區為準
- 營養素數據以每 100g 為基準單位儲存，依份量換算顯示
- 體重單位支援公斤（kg）與磅（lb）切換

## 3. 功能需求

### 3.1 使用者管理（MOD-USER）

#### FR-USER-001：註冊與登入

- 支援 Email + 密碼註冊
- 支援第三方登入（Google、Apple）
- 密碼需符合強度要求（至少 8 字元，包含大小寫與數字）

#### FR-USER-002：個人檔案設定

- 設定基本資料：姓名、性別、出生日期、身高
- 設定活動量等級（久坐 / 輕度活動 / 中度活動 / 高度活動 / 極高活動）
- 系統依據個人資料自動計算 BMR 與 TDEE

#### FR-USER-003：偏好設定

- 單位切換（公制 / 英制）
- 語言設定（繁體中文 / English）
- 通知提醒開關與時間設定

---

### 3.2 身體組成紀錄（MOD-BODY）

#### FR-BODY-001：體重紀錄

- 輸入日期、時間、體重值
- 支援手動輸入與智慧體重計資料匯入
- 每日可記錄多筆（如早晚量測）
- 顯示當日最新值為代表值

#### FR-BODY-002：體脂肪紀錄

- 輸入體脂率（%）
- 系統自動計算脂肪重量（kg）與去脂體重

#### FR-BODY-003：身體圍度紀錄

- 支援記錄：胸圍、腰圍、臀圍、上臂圍、大腿圍、小腿圍
- 各部位可獨立記錄日期與數值

#### FR-BODY-004：其他身體指標

- 骨骼肌肉量（kg）
- 內臟脂肪等級
- 身體水分率（%）
- 基礎代謝率（自動計算或手動輸入）

#### FR-BODY-005：身體組成趨勢圖

- 折線圖顯示各指標隨時間變化
- 支援選擇時間範圍（7 天 / 30 天 / 90 天 / 1 年 / 全部）
- 支援多指標疊加比較
- 顯示移動平均線以平滑波動

---

### 3.3 飲食紀錄（MOD-DIET）

#### FR-DIET-001：餐次紀錄

- 每日分為：早餐、午餐、晚餐、點心/宵夜
- 使用者可自訂餐次名稱

#### FR-DIET-002：食物搜尋與選擇

- 內建食物資料庫（含台灣常見食物）
- 支援關鍵字搜尋
- 顯示最近使用與常用食物
- 支援條碼掃描查詢包裝食品

#### FR-DIET-003：食物份量設定

- 輸入重量（公克）或選擇常見份量單位（碗、杯、片、個等）
- 系統自動換算營養素含量

#### FR-DIET-004：自訂食物

- 使用者可新增自訂食物項目
- 輸入名稱與每 100g 營養素含量
- 自訂食物可標記分類（主食、肉類、蔬菜、水果、飲料等）

#### FR-DIET-005：食譜 / 組合餐

- 可建立食譜（多項食物組合）
- 食譜可設定份數，自動計算每份營養素
- 食譜可重複使用快速記錄

#### FR-DIET-006：每日營養素摘要

- 顯示每日攝取總熱量（kcal）
- 顯示巨量營養素比例（碳水 / 蛋白質 / 脂肪）
- 以圓餅圖或環形圖呈現
- 與每日目標值比較，顯示達成率

#### FR-DIET-007：飲水紀錄

- 記錄每次飲水量（ml）
- 顯示每日飲水進度條
- 可設定每日飲水目標

---

### 3.4 運動紀錄（MOD-EXERCISE）

#### FR-EXERCISE-001：運動項目選擇

- 內建運動資料庫（有氧、重訓、球類、伸展等）
- 支援搜尋與分類瀏覽
- 支援自訂運動項目

#### FR-EXERCISE-002：有氧運動紀錄

- 記錄運動類型、持續時間、距離（選填）
- 系統估算消耗熱量（基於體重與運動強度）

#### FR-EXERCISE-003：重量訓練紀錄

- 記錄動作名稱、組數、每組次數、重量
- 支援記錄 RPE（自覺疲勞度）
- 自動計算訓練量（Volume = 組數 × 次數 × 重量）

#### FR-EXERCISE-004：運動歷史與統計

- 依日期 / 運動類型查詢歷史紀錄
- 統計每週運動頻率與時間
- 重訓進度追蹤（各動作最大重量趨勢）

---

### 3.5 健康指標追蹤（MOD-HEALTH）

#### FR-HEALTH-001：血壓紀錄

- 記錄收縮壓、舒張壓、脈搏
- 標註量測時段（晨間 / 日間 / 睡前）
- 依標準顯示血壓分級（正常 / 偏高 / 高血壓）

#### FR-HEALTH-002：血糖紀錄

- 記錄血糖值（mg/dL）
- 標註量測時機（空腹 / 餐前 / 餐後 2 小時）
- 依標準顯示正常 / 偏高 / 高血糖

#### FR-HEALTH-003：睡眠紀錄

- 記錄入睡時間、起床時間
- 自動計算睡眠時長
- 選填睡眠品質評分（1-5 分）

#### FR-HEALTH-004：自訂指標

- 使用者可新增自訂追蹤指標
- 設定指標名稱、單位、正常範圍
- 如：膽固醇、尿酸、HbA1c 等

---

### 3.6 目標設定與追蹤（MOD-GOAL）

#### FR-GOAL-001：體重目標

- 設定目標體重與預期達成日期
- 系統計算每週建議減重 / 增重速率
- 自動調整每日熱量建議

#### FR-GOAL-002：營養素目標

- 設定每日熱量目標（可手動或依 TDEE 自動建議）
- 設定巨量營養素目標比例或克數
- 設定飲水量目標

#### FR-GOAL-003：運動目標

- 設定每週運動天數目標
- 設定每週運動時間目標

#### FR-GOAL-004：進度追蹤儀表板

- 以進度條 / 環形圖顯示各目標達成率
- 每日 / 每週摘要通知
- 連續達標天數（Streak）統計

---

### 3.7 數據分析與報表（MOD-REPORT）

#### FR-REPORT-001：每日摘要

- 總覽當日：攝取熱量 vs 消耗熱量、營養素達成率、運動紀錄、身體指標

#### FR-REPORT-002：週報 / 月報

- 各指標平均值與趨勢
- 目標達成率統計
- 與上一期比較

#### FR-REPORT-003：長期趨勢分析

- 體重 / 體脂變化與飲食熱量的相關性圖表
- 運動量與身體組成變化對照

#### FR-REPORT-004：資料匯出

- 支援匯出 CSV 格式
- 支援匯出 PDF 報表
- 可選擇匯出日期範圍與資料類別

---

### 3.8 通知與提醒（MOD-NOTIFY）

#### FR-NOTIFY-001：定時提醒

- 提醒記錄體重（每日指定時間）
- 提醒記錄飲食（依餐次時間）
- 提醒喝水（可設定間隔）

#### FR-NOTIFY-002：目標提醒

- 當日熱量即將超標時警示
- 每週運動目標未達標提醒

## 4. 非功能需求

### 4.1 效能需求

| 項目 | 要求 |
|------|------|
| API 回應時間 | 一般查詢 < 500ms，報表查詢 < 2s |
| 頁面載入時間 | 首次載入 < 3s（LTE 網路環境） |
| 並發使用者 | 支援至少 1,000 同時在線 |

### 4.2 安全性需求

- 密碼使用 bcrypt 雜湊儲存
- API 使用 JWT 認證，Token 有效期 24 小時
- 敏感健康資料傳輸全程 HTTPS 加密
- 符合個人資料保護法規範

### 4.3 可用性需求

- 系統可用率 ≥ 99.5%
- 資料每日自動備份，保留 30 天
- 支援資料匯出供使用者自行備份

### 4.4 可維護性

- 前後端分離架構
- API 版本控制（v1, v2...）
- 完整的自動化測試覆蓋（單元測試 ≥ 80%）

### 4.5 相容性

- 瀏覽器：Chrome、Firefox、Safari、Edge 最新兩個主要版本
- 行動裝置：iOS 15+、Android 12+ 之內建瀏覽器
- 螢幕寬度：最小支援 320px

## 5. 資料模型

### 5.1 核心實體

```
User
├── id (PK)
├── email (UNIQUE)
├── password_hash
├── name
├── gender
├── birth_date
├── height_cm
├── activity_level
├── unit_preference (metric / imperial)
├── language
├── created_at
└── updated_at

BodyRecord
├── id (PK)
├── user_id (FK → User)
├── recorded_at (DATETIME)
├── weight_kg
├── body_fat_pct
├── muscle_mass_kg
├── visceral_fat_level
├── body_water_pct
├── bmr_kcal
├── chest_cm
├── waist_cm
├── hip_cm
├── upper_arm_cm
├── thigh_cm
└── calf_cm

Food
├── id (PK)
├── name
├── category
├── calories_per_100g
├── carbs_g
├── protein_g
├── fat_g
├── fiber_g
├── sodium_mg
├── barcode (NULLABLE)
├── is_custom (BOOLEAN)
└── created_by (FK → User, NULLABLE)

MealLog
├── id (PK)
├── user_id (FK → User)
├── date
├── meal_type (breakfast / lunch / dinner / snack)
├── food_id (FK → Food)
├── quantity_g
├── calories_kcal (計算值)
└── note

WaterLog
├── id (PK)
├── user_id (FK → User)
├── logged_at (DATETIME)
└── amount_ml

ExerciseLog
├── id (PK)
├── user_id (FK → User)
├── date
├── exercise_name
├── exercise_type (cardio / strength / flexibility)
├── duration_min
├── distance_km (NULLABLE)
├── sets (NULLABLE, JSON: [{reps, weight_kg, rpe}])
├── calories_burned
└── note

HealthRecord
├── id (PK)
├── user_id (FK → User)
├── recorded_at (DATETIME)
├── type (blood_pressure / blood_sugar / sleep / custom)
├── value_json (JSON: 依類型存放不同欄位)
└── note

Goal
├── id (PK)
├── user_id (FK → User)
├── type (weight / calories / macro / water / exercise)
├── target_value
├── target_unit
├── deadline (NULLABLE)
├── is_active (BOOLEAN)
└── created_at

Recipe
├── id (PK)
├── user_id (FK → User)
├── name
├── servings
├── items (JSON: [{food_id, quantity_g}])
└── created_at
```

### 5.2 ER 關係摘要

- User 1 ─ N BodyRecord

- User 1 ─ N WaterLog
- User 1 ─ N ExerciseLog
- User 1 ─ N HealthRecord
- User 1 ─ N Goal
- User 1 ─ N Recipe
- Food 1 ─ N MealLog

## 6. 使用者介面概述

### 6.1 主要頁面

| 頁面 | 說明 |
|------|------|
| 儀表板（Dashboard） | 當日摘要：體重、熱量攝取/消耗、目標進度 |
| 身體紀錄頁 | 輸入/瀏覽身體組成數據，趨勢圖表 |
| 飲食紀錄頁 | 按餐次記錄食物，查看營養素分析 |
| 運動紀錄頁 | 記錄運動內容，查看訓練歷史 |
| 健康指標頁 | 血壓 / 血糖 / 睡眠等指標紀錄 |
| 報表頁 | 週報 / 月報 / 趨勢分析 |
| 個人設定頁 | 個人資料、目標設定、偏好設定 |

### 6.2 導航結構

```
底部導航列（Mobile）/ 側邊導航列（Desktop）
├── 🏠 儀表板
├── 📊 身體紀錄
├── 🍽️ 飲食紀錄
├── 🏃 運動紀錄
├── ❤️ 健康指標
└── ⚙️ 設定
```

全域浮動按鈕（FAB）：快速新增紀錄（體重 / 飲食 / 運動 / 飲水）

## 7. 外部介面需求

### 7.1 第三方整合（未來規劃）

| 服務 | 用途 |
|------|------|
| 食品營養資料庫 API | 擴充食物資料庫 |
| 智慧體重計 SDK | 自動同步體重數據 |
| Google Fit / Apple Health | 同步運動與健康數據 |
| 條碼查詢 API | 掃描包裝食品條碼查詢營養資訊 |

### 7.2 API 設計原則

- RESTful 風格，資源導向
- JSON 格式請求與回應
- 統一錯誤回應格式：`{ "error": { "code": "...", "message": "..." } }`
- 分頁查詢使用 `?page=1&per_page=20`
